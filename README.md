## 带宽大不代表速度快——这个逻辑要理清

很多人搜搬瓦工带宽，其实真正想问的是"速度到底行不行"。这两件事要分开说。

带宽是管道的粗细。速度是你实际跑出来的数据量。影响速度的，除了带宽，还有线路质量——也就是你的数据从出发到目的地走的是哪条路。

搬瓦工的线路主要分几类：

**普通线路**（KVM 常规方案）：走公共互联网路由，不专门针对中国大陆优化。高峰期会拥堵，速度波动大。

**CN2 GT 线路**：电信的 CN2 骨干网，比普通线路稳定，但高峰期仍会有一定绕路。

**CN2 GIA 线路**：这个才是搬瓦工对国内用户的核心卖点。三网回程全程 CN2 GIA，直连、低延迟、丢包少。哪怕带宽"只有" 1Gbps，实测跑起来体感也比普通线路的 1Gbps 顺畅得多。

CN2 GIA-E 套餐把 CN2 GIA 线路和 2.5Gbps ~ 10Gbps 的带宽结合起来，就是这个系列价格高的原因——不是在卖带宽数字，是在卖线路优先级加带宽组合。

## 带宽超了会怎样？流量和带宽的关系

顺便说一个常被混淆的点：**带宽和流量不是一回事**。

- **带宽**：单位时间内能跑多快（速度上限）
- **月流量**：一个月内总共能传多少数据（总量上限）

搬瓦工各套餐都有月流量限额，从 500GB 到 12TB 不等。流量用满之后，带宽会被临时限速（降到很低），等下个月重置。

不过实际用下来，个人建站或者轻量代理用途，基本用不满 1TB/月 的流量配额。流量超用这个问题，通常是在跑大规模下载或者高并发视频流的场景才会碰到。
